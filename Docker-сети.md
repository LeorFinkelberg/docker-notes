### User-defined networks

Можно создать пользовательскую сеть и связать с помощью этой сети несколько контейнеров. После подключения к пользовательской сети контейнеры могут взаимодействовать друг с другом через свои IP-адреса или имена. 

В приведенном ниже примере с помощью драйвера сети `bridge` создается сеть и в созданной сети запускается контейнер
```bash
$ docker network create -d bridge my-net
$ docker run --network=my-net -itd --name=containter3 busybox
```
### Drivers

Перечисленные ниже драйверы сети доступны по умолчанию и предоставляют основную функциональность сетевого взаимодействия:
- `bridge`: драйвер сети по умолчанию.
- `host`: удаляет сетевую изоляцию между контейнером и Docker-хостом.
- `none`: полностью изолирует контейнер от Docker-хоста и других контейнеров.
- `overlay`: соединяет несколько Docker-демонов вместе.
- `ipvlan`: IPvlan-сети обеспечивают полный контроль как над IPv4-, так и над IPv6-адресацией.
- `macvlan`: назначает контейнеру MAC-адрес.

### Published ports

По умолчанию, когда вы создаете или запускаете контейнер, использую команды `docker create` или `docker run`, _контейнеры не открывают (doesn't expose) свои порты наружу_. Для того чтобы сделать порт доступным для сервисов вне Docker, следует использовать флаг `--publish` (или короткий вариант `-p`). Это создаст правило на хосте, отображающее порт контейнера на порт хоста.

Примеры:
- `-p 8080:80` отображает порт `8080` на хосте на порт `80` контейнера (используется TCP).
- `-p 192.168.1.100:8080:80` отображает порт `8080` на хосте с IP-адресом `192.168.1.100` на порт `80` контейнера (используется TCP).
- `-p 8080:80/udp` отображает порт `8080` на хосте на порт `80` контейнера (используется UDP).
- `-p 8080:80/tcp -p 8080:80/udp` отображает TCP-порт `8080` на хосте на TCP-порт `80` контейнера, а также отображает UDP-порт `8080` на хосте на UDP-порт `80` контейнера.

Для того чтобы контейнеры могли взаимодействовать друг с другом нет необходимости публиковать (`-p 8080:80`) их порты. Можно организовать межконтейнерное взаимодействие через Docker-сеть (обычно это мостовая сеть).
### IP address and hostname

Хост `0.0.0.0` означает шаблон "все сетевые интерфейсы".

По умолчанию контейнеры получают IP-адрес для каждой Docker-сети, к которой они подключены. Каждая сеть по умолчанию имеет маску подсети и шлюз. Запущенный контейнер можно подключить к нескольким сетям.
